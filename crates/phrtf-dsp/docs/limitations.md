# 制限と今後の改善

## 1. pHRTFは正中面モデル

飯田氏らのpHRTFは、正中面の垂直定位に強いモデルです。本実装ではそれを任意3D方向に拡張していますが、真の全天球HRTFではありません。

不足しているもの:

- 方位角ごとのP/Nパラメータ
- 左耳/右耳で異なる実測PNP
- 下半球の実測PNP
- 肩・胴体・頭部回折の詳細
- 距離依存HRTF、near-field HRTF

## 2. 実測HRTFと一致しない

デフォルトの `PhrtfProfile` は平均的な初期値であり、個人に一致しません。

実用化するなら以下を行ってください。

1. RIEC/CIPIC/自前測定HRTFからP/Nを抽出
2. 方向ごとに `PhrtfBandSet` を作る
3. 角度間を補間
4. N1/N2の中心周波数、gain、Qを個人化
5. 実測HRTFとのLog Spectral Distortionを最小化

## 3. 空気吸収EQは近似

ISO 9613-1の式は周波数ごとに連続的な損失曲線を返します。本実装はそれを3バンド程度のIIRで近似しています。

もっと厳密にするなら:

- 1/3 octave bandごとにフィットする
- 高域にもっと多くのピーキングEQを置く
- FIRを設計する
- 距離ごとに最小二乗でbiquadパラメータを最適化する

## 4. 近接効果は演出モデル

本実装の近接効果は、指向性マイクの近接効果に基づくlow-shelf boostです。

人間の耳に対する本当の近距離HRTFではありません。近距離HRTFをやるなら、距離別HRTF測定データや球面波HRTFモデルが必要です。

## 5. 係数更新のzipper noise

`update()` は係数を即時更新します。音源が連続移動する場合は、以下のどちらかを入れてください。

- 方向・距離パラメータを平滑化してから `update()`
- 旧フィルタ/新フィルタを短時間クロスフェード
- biquad係数をサンプル単位またはブロック単位で補間

## 6. ゲーム実装のおすすめ

- 直接音だけこのpHRTFで処理
- 反射/残響は別バスで距離に応じて増やす
- 遠距離では直接音を減らし、残響/環境音比を上げる
- very closeでは近接効果とコンプレッションを足す
- HRTFスペクトル強度はユーザー設定で調整可能にする
